1 · Editorial Instrument

Financial-broadsheet authority meets a precision instrument. Surprise through restraint and craft.

Typography

Fraunces / GT Sectra (display serif) · Söhne / Neue Haas (UI) · Söhne Mono (tabular numerals everywhere). Light serif label over heavy value.

Depth

No shadows — hairline rules + whitespace + faint ledger-grid texture. One hero metric per page on a paper gradient with a scarlet edge rule.

Motion

Page "assembles to print": title rises, KPIs reveal left-to-right (60ms stagger) with odometer count-up, charts draw their stroke. Approve = letterpress "APPROVED" stamp.

Risk

Needs licensed fonts + typographic discipline. Fallback: Fraunces + Geist (open).

2 · Neo-Terminal Theming System

A complete, extensible CSS system — light/dark via data-theme class toggle, WCAG AA verified, distinctive cyberpunk-tempered palette, new themes drop in with no refactor.

Architecture

3-tier tokens — primitives → semantic (theme-switched) → component. Components read only semantic tokens, so a theme is just a values block.

Accessibility

Every text/bg pairing + accent-contrast meets AA (≥4.5:1 body, ≥3:1 UI). Ships with a contrast lint in CI — it's a test, not a suggestion.

Extensibility

Add [data-theme="rvgt-classic"] or [data-theme="client-acme"] as one block; @layer lets client overrides avoid specificity wars.

Risk

Lowest-risk, most reusable. Maps cleanly onto the existing brand.py token approach.

3 · Mission Control

Borrows game feedback systems (not aesthetics): celebration on real wins, progressive rewards as earned depth + operating streaks. Best fit for Apex's directive/approval loop.

Mechanic 1 — Consequence feedback

Celebrate only objectively-good, rare events (KPI hits target, directive completes, scenario beats plan): ≤600ms flourish, ring completion, over-shoot count-up. Never blocks, never on routine refresh.

Mechanic 2 — Progressive rewards

Operating streaks ("on-pace 6 weeks"), progressive disclosure of advanced levers, a mission tracker for multi-step flows (build → review → submit → approve → executed).

3 layouts

A — Command Console (Scorecard: HUD + radial gauge + alert wire + console input). B — Strategy Map (Spend/Funnel: resource nodes + particle flows + "commit plan" → directive). C — Mission Debrief (Simulator: pre/post briefing + scan-line compute + debrief).

Risk

Most motion/QA discipline. Rarity, brevity, data-truth, opt-in sound, reduced-motion = mandatory.
